# 变更提案: 独立视频平台与失败退款

## 需求背景
现有 Grok 视频功能绑定 OAuth 平台，无法把 `cpa-fan` 的 Base URL 与 API Key 作为独立视频账号池接入；异步任务失败后也缺少自动余额退款。

## 产品分析

### 目标用户与场景
- **用户群体:** 管理多个视频上游账号和散户 API Key 的运营管理员。
- **使用场景:** 配置视频账号、按分组售卖 480p/720p 视频额度、自动处理失败任务。
- **核心痛点:** 上游接入不独立、定价模式不完整、失败需要人工补款。

### 价值主张与成功指标
- 一个账号入口完成凭证、模型、分组与价格配置。
- 视频任务失败后无需人工查询或手工加余额。

### 人文关怀
不记录提示词和敏感凭证到任务表；退款失败保留可审计状态，避免用户资金损失。

## 变更内容
1. 新增独立 `video` 平台和完整 cpa-fan 视频路由。
2. 分组新增按秒/按次计费模式，复用 480p/720p 价格字段。
3. 创建账号时可快捷更新所选视频分组价格。
4. 持久化异步任务，后台自动查询并对失败任务执行余额退款。

## 影响范围
- **模块:** 账号、分组、网关、调度、计费、任务、管理端。
- **API:** 管理账号/分组 API 与视频网关 API。
- **数据:** groups 新字段与 video_tasks 新表。

## 核心场景

### 需求: 视频平台账号
**模块:** 账号管理

#### 场景: 管理员创建账号
- 可填写 Base URL、API Key、同步模型和绑定同平台分组。
- 账号只参与视频请求。

### 需求: 视频任务计费
**模块:** 分组与计费

#### 场景: 创建与失败
- 创建成功立即按分组规则扣余额并持久化任务。
- 后台读到失败终态时只退款余额，重复处理不重复退款。

## 风险评估
- **风险:** 数据库迁移、重复退款、轮询并发、上游暂时不可用。
- **缓解:** 先备份；事务条件更新与唯一键幂等；任务租约；网络错误重试而非退款。
