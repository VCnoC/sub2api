# Changelog

本文件记录项目的重要变更，格式遵循 Keep a Changelog。

## [Unreleased]

### Added
- 支持同一用户持有同分组的多张独立订阅，并按最早到期且当前可用的订阅自动消费。
- 新增普通与豪华双奖池邀请抽奖系统，支持每日/每周基础次数、长期额外次数、活动时间和准确概率展示。
- 新增注册、首次兑换、单笔/累计充值机会规则及退款冲正，支持同一规则同时奖励一个或两个奖池。
- 新增余额与订阅自动发奖、用户轮带抽奖页、个人历史及管理员奖池、奖品、规则和审计管理页。
- 新增站内工单系统，支持用户创建与双向回复、管理员统一队列、内部备注、可选负责人、优先级和关闭后用户回复自动重开。
- 新增工单私有附件、个人未读游标、60 秒侧边栏角标、邮件通知和关闭 30 天后的附件清理任务。
- 新增用户与管理员工单页面、完整中英文文案，以及附件、状态流转、并发限流和权限隔离测试。
- 新增独立 `video` 平台，支持使用 Base URL、API Key、模型同步和模型白名单管理 cpa-fan 视频上游账号。
- 新增 cpa-fan 兼容的视频创建、生成、编辑、延长和查询路由。
- 新增 `video_tasks` 持久化任务与数据库租约 worker，进程重启后可继续跟踪异步终态。
- 对话广场新增视频分组生成能力，支持文生视频、1.5 preview 单图生视频、0-100% 进度和原生播放器。
- 新增 JWT 鉴权的 `POST /api/v1/playground/videos` 与 `GET /api/v1/playground/videos/:request_id`。
- 对话广场新增 `gpt-image-2-vip` 生图模式，支持尺寸、质量、返回格式、背景、风格和水印参数选择。
- 新增 JWT 鉴权的 `POST /api/v1/playground/images/generations`，复用 OpenAI Images 计费与调度链路。

### Changed
- 合并官方 `v0.1.160`，接入管理操作审计、提示词审计、异步图片任务、step-up 2FA、上游倍率探测及网关/调度修复，同时保留本地 Grok、独立视频、工单、抽奖、多订阅和账号批量用量能力。
- 官方迁移保持 174-182，本地支付订单订阅关联与多订阅候选索引顺延为 183/184，避免迁移编号冲突。
- 订阅购买、兑换、抽奖和管理员发放改为创建独立权益；支付订单关联具体订阅以支持精确退款。
- 合并官方 `v0.1.153`，同步 Grok API Key 自定义 Base URL、上游模型同步、视频编辑/延长及相关网关修复，同时保留本地独立 `video` 平台。
- 抽奖管理端订阅奖品改为选择现有订阅套餐，并自动写入套餐对应的订阅分组和有效天数。
- 视频分组支持 `per_second` 和 `per_request` 两种计费模式；独立视频平台固定使用余额计费。
- 视频任务扣费与任务写入在同一幂等事务中完成。
- 默认 CSP 允许加载 HTTPS 视频媒体；视频状态查询作为已扣费任务的只读操作，不重复要求正余额。
- 对话广场切换到 `grok-imagine-video*` 时显示 1-15 秒时长和画面比例选择，并随创建请求发送。

### Fixed
- 抽奖次数、库存、权益和记录在同一事务内提交；重复事件、并发抽奖或发奖失败不会重复扣次、超发库存或留下部分权益。
- 修复共享 Axios 客户端强制使用 JSON Content-Type，导致无附件工单的 FormData 被序列化为 JSON 并误报附件无效的问题。
- 强化工单 WebP 附件校验，拒绝仅伪造 RIFF/WEBP 文件头的无效图片。
- 修复日期范围组件使用 Teleport 后测试仍在本地 wrapper 查找下拉项的问题。
- 上游明确返回失败终态时自动退回本次余额，并通过任务锁和退款时间保证重复处理不重复退款。
- 修复对话广场视频模式仍显示文档入口、参考图无法移除，以及余额恰好扣至零后无法继续查询任务的问题。
