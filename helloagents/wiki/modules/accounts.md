# 账号管理

## 目的
维护可调度的上游账号及其凭证、模型限制、代理和分组绑定。

## 规范

### 需求: 平台隔离
**模块:** 账号管理

#### 场景: 创建上游账号
- 平台、认证类型和凭证必须在服务端校验。
- 敏感凭证在管理 API 响应中脱敏。
- 账号只绑定同平台分组，混合绑定必须经过已有风险检查。
- `grok` 账号支持 OAuth 和 API Key；API Key 账号默认使用官方 xAI Base URL，也可配置经过校验的第三方 Base URL 和请求头覆写。
- `video` 账号固定使用 API Key 认证，凭证包含 Base URL 与 API Key，只参与视频路由。

### 需求: 上游模型同步
**模块:** 账号管理

#### 场景: 创建账号前同步
- 管理员可使用 Base URL 与 API Key 调用上游模型列表。
- 返回值只包含规范化、去重并排序后的模型 ID。
- Grok API Key 与独立 Video 账号分别使用自己的平台凭证同步模型，不跨平台调度。
- 上游没有返回模型时，视频平台使用内置 Grok 视频模型作为可选默认值。
- 管理端可批量读取账号用量，并使用官方上游倍率探测结果辅助核对 Grok 等账号的计费倍率。
- Grok OAuth 账号展示周计费快照或 Free 24 小时 token 用量；Free 上限以服务端动态字段为准，不使用前端硬编码。
- 账号列表继续由父页面批量加载用量；手动 Grok 探测只覆盖当前单元格结果，不恢复逐账号首屏请求。

### 需求: 账号列表批量统计
**模块:** 账号管理

#### 场景: 加载当前窗口费用
- 同一页账号的窗口费用使用单次 PostgreSQL 批量聚合查询。
- 查询次数不随开启窗口限额的账号数量线性增长。
