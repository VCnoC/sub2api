# 变更提案: 对话广场视频生成

## 需求背景
独立视频平台已经支持视频任务创建、查询、按秒或按次计费以及失败退款，但对话广场仍只会调用 Chat Completions，选择视频分组后无法使用这些能力。

## 产品分析

### 目标用户与场景
- **用户群体:** 已登录站点、希望直接在网页中使用 Grok 视频模型的余额用户。
- **使用场景:** 在对话广场选择视频分组和模型，通过文本或一张参考图生成并播放视频。
- **核心痛点:** 用户必须自行创建 API Key 并调用异步接口，网页内没有任务进度和结果展示。

### 价值主张与成功指标
- **价值主张:** 复用已有账号、计费和退款链路，在对话广场完成视频生成闭环。
- **成功指标:** 视频任务可创建、可查询终态，成功结果可播放，明确失败可显示错误并由现有机制退款。

### 人文关怀
生成过程提供明确状态，失败不隐藏上游错误；附件继续只随用户主动提交的请求和会话保存，不引入新的数据采集。

## 变更内容
1. 为对话广场增加 JWT 鉴权的视频创建与状态查询入口。
2. 视频分组发送时改走异步视频接口，支持文本和单张参考图。
3. 在消息中持久化任务状态并使用浏览器原生播放器展示结果。

## 影响范围
- **模块:** Playground、视频网关、前端消息渲染。
- **文件:** Playground 路由/中间件、前端 API/类型/发送与消息组件。
- **API:** 新增 `POST /api/v1/playground/videos` 与 `GET /api/v1/playground/videos/:request_id`。
- **数据:** 不新增数据表；视频结果随现有会话 messages JSON 保存。

## 核心场景

### 需求: 对话广场视频生成
**模块:** playground-video
用户在视频分组中发起并查看视频生成。

#### 场景: 文生视频
选择普通视频模型并输入提示词。
- 创建视频任务并显示生成进度。
- 成功后在助手消息内播放视频。

#### 场景: 图生视频
选择 1.5 视频模型并上传一张图片。
- 以 `input_reference.image_url` 提交参考图。
- 未上传图片时在发送前给出明确错误。

#### 场景: 失败与中断
上游返回失败或用户停止等待。
- 上游失败信息展示在消息中，余额退款由既有视频任务机制处理。
- 用户停止仅终止浏览器轮询，不重复提交或触发退款。

## 风险评估
- **风险:** GET 状态查询没有请求体，无法沿用原中间件读取分组。
- **缓解:** 仅为 GET 从 query 读取 group，其他请求保持原 JSON 校验语义。
- **风险:** Data URL 体积较大。
- **缓解:** 继续使用现有 10MB 单图片限制和网关请求体上限，不增加副本存储。
