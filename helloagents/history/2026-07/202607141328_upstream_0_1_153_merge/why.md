# 变更提案: 合并官方 v0.1.153

## 需求背景
本地分支在官方版本基础上增加了独立视频平台、对话广场、工单及抽奖等能力，需要吸收官方 v0.1.153 的修复和 Grok 能力，同时保持本地 `video` 平台边界。

## 变更内容
1. 合并官方标签 `v0.1.153`。
2. 在共享媒体路由和账号管理中同时保留 `grok` 与 `video` 分支。
3. 验证后端生成代码、媒体路由和前端账号表单。

## 影响范围
- **模块:** 网关媒体、账号管理、上游模型同步、前端账号表单
- **文件:** 官方合并产生的变更及人工解决的冲突文件
- **API:** `/v1/videos/generations`、`/v1/videos/edits`、`/v1/videos/extensions`、`/v1/videos/:request_id`
- **数据:** 保留本地 `video` 平台和官方 v0.1.153 数据迁移

## 核心场景

### 需求: Grok 与独立视频平台并存
**模块:** 网关媒体

#### 场景: 按分组平台转发视频请求
API Key 绑定 `grok` 或 `video` 分组时：
- `grok` 请求使用官方 Grok 账号能力。
- `video` 请求使用本地独立视频上游、余额计费和任务持久化。

### 需求: 保留本地定制
**模块:** 账号管理

#### 场景: 合并官方账号能力
管理端创建或编辑账号时：
- 保留官方 Grok API Key、Base URL 和模型同步能力。
- 保留本地 Video API Key、Base URL 和模型同步能力。

## 风险评估
- **风险:** 自动合并可能在共享路由中覆盖某一平台分支，生成代码也可能与 Schema 不一致。
- **缓解:** 在独立分支执行，人工逐项合并条件分支，重新生成 Wire/Ent 并运行关键测试。
