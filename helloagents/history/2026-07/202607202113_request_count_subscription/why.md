# 变更提案: 纯次数订阅套餐

## 需求背景
现有订阅分组只支持按 USD 用量设置日、周、月限额，无法表达“成功文本调用扣 1 次”的套餐。API Key 虽已有 5 小时、1 天窗口，但其限制仍按 USD 计算且作用于单个 Key，用户可通过多个 Key 绕过，因此不能作为订阅权益。

本功能沿用现有“创建订阅分组，再创建套餐绑定分组”的产品流程，不新增独立次卡管理入口。

## 产品分析

### 目标用户与场景
- **用户群体:** 需要销售固定调用次数套餐的站点管理员，以及购买该套餐的 API 用户。
- **使用场景:** 管理员创建纯次数订阅分组，配置 5 小时和 24 小时文本调用上限，再创建普通订阅套餐绑定该分组。
- **核心痛点:** Token 成本波动使套餐难以用固定 USD 额度描述，且现有 Key 级限制不具备订阅权益隔离能力。

### 价值主张与成功指标
- **价值主张:** 使用与现有订阅一致的购买、兑换、多卡和到期体验，提供容易理解的固定次数权益。
- **成功指标:** 并发请求不超过上限；失败请求最终不占次数；每张订阅独立计数；现有 USD 套餐行为不变。

### 人文关怀
用户端明确展示已用、剩余和重置时间；限额错误不暴露内部账号或调度信息。临时占位只保存请求标识和权益关联，不保存提示词正文。

## 变更内容
1. 订阅分组增加 `usd` 与 `request_count` 两种计费模式；次数模式配置 5 小时和 24 小时文本请求上限。
2. 每张用户订阅保存独立的次数窗口与计数，多张同组订阅继续按最早到期顺序消费。
3. 文本请求发送上游前在 PostgreSQL 原子占位；成功后确认，明确失败或切组时释放。
4. 次数模式不检查或累加订阅 USD 用量，但保留使用日志中的成本数据用于运营统计。
5. 管理端和用户端展示次数套餐配置、已用次数、剩余次数和重置时间。

## 影响范围
- **模块:** 分组管理、订阅权益、API Key 多分组容灾、文本 Gateway、统一计费、订阅进度、支付套餐展示。
- **文件:** Ent schema、SQL migration、订阅 Repository/Service、计费事务、Gateway/Handler、Vue 管理页与用户页、测试和知识库。
- **API:** 分组创建/更新/列表、套餐分组预览、订阅进度响应增加次数模式字段。
- **数据:** `groups`、`user_subscriptions` 新字段；新增请求占位表。

## 核心场景

### 需求: 创建纯次数订阅分组
**模块:** 分组与套餐管理

#### 场景: 管理员创建套餐
管理员创建订阅分组并选择按次数计费。
- 可配置 5 小时与 24 小时文本调用上限，正整数表示启用，0 表示该窗口不限。
- 创建套餐时仍只选择该分组，不在套餐表重复保存次数配置。
- 次数模式隐藏 USD 日、周、月限额；USD 模式行为保持不变。

### 需求: 独立次数权益
**模块:** 订阅与计费

#### 场景: 首次成功文本调用
用户持有次数套餐并发起成功文本请求。
- 首次占位同时启动连续 5 小时和 24 小时窗口。
- 一个逻辑请求只占用 1 次，内部同账号重试和换账号不重复占位。
- 成功后占位转为已确认次数，不扣订阅 USD 额度。

#### 场景: 调用失败
请求校验失败、上游超时或上游返回 4xx/5xx。
- 释放本次占位，5 小时和 24 小时计数均恢复。
- API Key 跨组容灾前先释放旧组占位，再为新组重新占位。
- 客户端主动中断但上游已经接受并开始返回内容时，确认本次次数，防止利用中断绕过限制。

#### 场景: 次数耗尽
任一启用窗口的已占用次数达到上限。
- 当前订阅不能继续占位，尝试同组下一张最早到期订阅。
- 同组全部耗尽后按现有多分组容灾规则尝试下一组。
- 没有其他可用权益时返回明确的 429 次数耗尽错误及最近重置时间。

### 需求: 仅文本请求计次
**模块:** Gateway

#### 场景: 请求分类
- Messages、Chat Completions、Responses 和 Gemini 文本生成的同步、流式请求参与计次。
- `count_tokens`、模型列表、用量查询、图片、视频、批量任务及任务状态查询不计次。
- Responses 或 Gemini 路由识别为图片/视频意图时不创建次数占位。

## 风险评估
- **风险:** 并发选卡和占位竞态导致超发。**缓解:** PostgreSQL 事务锁定具体订阅行，并在同一事务内重置窗口、检查上限、增加占用和写入幂等占位记录。
- **风险:** 进程异常留下 pending 占位。**缓解:** 占位带过期时间，下一次占位时在同一订阅内顺带回收过期记录；窗口自然到期也会清除旧计数。
- **风险:** 成功响应与次数确认状态不一致。**缓解:** 成功确认加入现有 `usage_billing` 幂等事务；失败释放为幂等状态转换。
- **风险:** 路由误把媒体请求算作文本。**缓解:** 复用现有 endpoint 归一化和图片/视频意图判断，不单靠 URL 前缀。
