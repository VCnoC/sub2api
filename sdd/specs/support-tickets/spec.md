# Specification: 站内工单系统 (Specification)

## 1. Scope
- **In Scope**: 登录用户工单、管理员统一处理、双向公开回复、内部备注、可选负责人、状态/分类/优先级、私有附件、站内未读、邮件通知、创建限流和关闭后附件清理。
- **Out of Scope**: 游客工单、外部客服角色、外部工单 SaaS、邮件直接回复、实时 WebSocket、调用记录自动关联、消息编辑删除、用户自助关闭、自动超时关闭、满意度评价、病毒扫描和全文附件搜索。

## 2. Functional Requirements

### 2.1 用户创建与查看
- **Trigger**: 已登录用户从侧边栏进入“工单”并创建工单。
- **UI/UX**: 必填标题、分类和正文；可附加图片或文本文件。列表显示编号、标题、分类、状态和最后更新时间。
- **Logic**: 用户只能查看自己的工单；新工单默认 `pending_admin` 和 `normal`。每位用户最多同时存在 5 个未关闭工单，每个自然日最多创建 10 个。

### 2.2 分类与优先级
- **Trigger**: 用户创建工单或管理员处理工单。
- **UI/UX**: 用户选择账号、充值计费、API 调用、模型故障或其他；管理员可设置普通、高或紧急。
- **Logic**: 用户不能设置优先级；新工单默认普通。管理员列表可按分类和优先级筛选，并优先展示紧急、高优先级及更早待处理的工单。

### 2.3 双向回复与关闭
- **Trigger**: 用户或管理员在工单详情发送回复，或管理员关闭工单。
- **UI/UX**: 公开消息按时间显示；关闭工单仍保留回复框，明确提示用户回复会重新打开。
- **Logic**: 用户回复后状态为 `pending_admin`；管理员公开回复后状态为 `pending_user`；仅管理员可关闭。用户回复已关闭工单时自动重开，不进行自动超时关闭。

### 2.4 内部备注与负责人
- **Trigger**: 管理员添加内部备注、认领或转交工单。
- **UI/UX**: 内部备注与公开回复视觉区分，并明确标记仅管理员可见；负责人可为空。
- **Logic**: 所有管理员始终可查看和回复；负责人不形成独占权限。内部备注不改变状态且不通知用户。负责人和优先级变化必须保留管理员可见审计事件。

### 2.5 附件
- **Trigger**: 用户或管理员发送公开回复时选择文件。
- **UI/UX**: 上传前显示文件名、大小、移除操作和整体上传状态；已过期附件显示“已清理”元数据，不显示失效下载链接。
- **Logic**: 允许 JPG、PNG、WebP、TXT、LOG、JSON；每条回复最多 5 个文件，单文件最多 5 MB，总计最多 20 MB。附件只能由工单所属用户或管理员鉴权访问。

### 2.6 附件保留与删除
- **Trigger**: 管理员关闭工单、用户重开工单、后台清理到期附件或管理员手动删除附件。
- **UI/UX**: 文字记录长期保留；附件被清理后保留名称、大小和删除时间。
- **Logic**: 工单关闭时为未删除附件设置 30 天删除时间；30 天后物理删除文件但保留元数据。关闭前重开则取消未到期附件的删除计划；已经删除的附件不恢复。

### 2.7 站内未读
- **Trigger**: 对方发送公开回复、新工单创建、内部备注或管理员可见状态事件产生。
- **UI/UX**: 用户和管理员侧边栏显示未读工单角标；打开详情后仅清除当前查看者对该工单的未读状态。
- **Logic**: 用户未读不包含内部备注和管理员私有事件；管理员未读按管理员个人维护。角标在登录后、窗口重新获得焦点和页面可见期间定时刷新。

### 2.8 邮件通知
- **Trigger**: 新工单、用户公开回复、管理员公开回复、关闭或负责人变更。
- **UI/UX**: 邮件只提供摘要和站内链接，不支持直接回复邮件。
- **Logic**: 新工单和未认领工单的用户回复通知全部正常管理员；认领后仅邮件通知负责人。管理员公开回复及关闭通知工单用户。内部备注不通知用户。邮件发送失败不能使工单操作失败。

### 2.9 内容不可变与管理删除
- **Trigger**: 消息发送成功或管理员发现违规/敏感附件。
- **UI/UX**: 不提供消息编辑和删除入口；管理员可删除附件并填写原因。
- **Logic**: 用户和管理员都不能修改或删除已发送正文。管理员删除附件后保留删除人、原因、时间和管理员可见事件。

### 2.10 管理员列表与筛选
- **Trigger**: 管理员进入“工单管理”。
- **UI/UX**: 支持状态、分类、优先级、负责人（全部、我的、未分配）和关键词筛选；默认先显示待管理员回复，再按优先级和等待时间排序。
- **Logic**: 关键词只搜索工单编号、标题和创建用户账号信息，不搜索附件内容；列表分页并保持筛选参数。

## 3. Acceptance Checklist
- [x] 登录用户可创建并分页查看自己的工单，无法读取其他用户的工单或附件
- [x] 分类、普通/高/紧急优先级和管理员筛选排序符合规格
- [x] 用户与管理员可连续公开回复，状态在待管理员和待用户之间正确切换
- [x] 仅管理员可关闭，用户回复已关闭工单时自动重开，系统不会自动关闭工单
- [x] 内部备注只对管理员可见且不触发用户通知
- [x] 负责人可选、可转交且不阻止其他管理员处理
- [x] 未认领和已认领工单的管理员邮件收件人规则正确
- [x] 用户收到管理员公开回复和关闭通知，邮件链接可回到对应工单
- [x] SMTP 或邮件模板失败不会回滚工单创建、回复或关闭
- [x] 每位用户最多 5 个未关闭工单、每天最多创建 10 个，并发请求无法绕过限制
- [x] 每条消息最多上传 5 个允许类型文件，单个 5 MB、总计 20 MB 的边界校验正确
- [x] 伪造扩展名、错误 MIME、路径穿越、无权下载和超限上传均被拒绝
- [x] 工单关闭 30 天后附件文件被幂等清理，文字和附件元数据保留
- [x] 工单重开会取消未到期附件的清理计划，但不尝试恢复已删除文件
- [x] 消息正文不可编辑删除，管理员删除附件会留下审计信息
- [x] 用户和每位管理员的未读角标独立且只统计其有权查看的消息
- [x] 用户页和管理员页在桌面与移动端无重叠、溢出或不可操作控件
- [x] 后端单元/集成测试、前端组件测试、TypeScript、ESLint 和生产构建通过
- [ ] 数据库迁移、持久化目录权限、附件备份和生产健康检查通过
